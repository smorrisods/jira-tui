Design proposal · docs/design · portrait companion set

jira-tui · UI refresh — when the terminal is taller than it is wide

Same design system as the landscape set — same palette, same chips, same maple selection bar — but re-composed for a tall, narrow terminal (a half-screen split, a vertical monitor, a side pane). The rule of the set: nothing is removed, everything restacks. Side rails become strips, the detail rail folds into a facts panel, and the board turns into a column pager. The proposed breakpoints are measured in terminal cells, not pixels:

wide ≥ 110 cols · side-by-side rails, 4-up board narrow < 90 cols · this set — stacked, paged board 60–110 cols · drop columns one at a time (assignee → type → updated)

1 · Welcome — stacks, doesn't shrink

      
Cards go vertical
The three onboarding cards stack full-width instead of sitting three across. Reading order is unchanged; the recommended maple border still answers "which one do I press?"

      
Jax is size-independent
The welcome Jax is 5 rows tall — he fits any terminal the app itself fits, so he's identical in both sets. Same blink, same 🍁/🍂 swap.

    


    

      
2 · Home — rails become strips

      
Context: one line, not a card stack
Repo, branch, and linked issue read fine as a single wrapped strip. The 32-column side rail only earns its width when the terminal has width to spare.

      
Glance stats: 4 tiles in a row
The stat list becomes four compact tiles with the same proportion bars. Same data, one text row taller than a plain line each.

      
Two-line rows for the selected issue
Narrow lists drop the type and assignee columns; the selected row grows a second line carrying exactly what was dropped (type chip, assignee, parent). Detail on demand, no horizontal squeeze.

      
Mini-Jax perches in every footer
A floating companion would cover real content at this width, so Jax docks as ●‿● jax 👋 in the footer of every screen (Welcome keeps the big Jax). He cycles his scenes right there in miniature; click him — he pops out, parties, and tucks back in. Happiness preserved at all breakpoints.

    


    

      
3 · List + quick view — vertical split stays

      
Column drop order is a rule, not a scramble
As width shrinks the list sheds columns in a fixed order: assignee → type → updated. Key, status chip, and summary are load-bearing and never drop. The header row shows only what's present.

      
Quick view: meta goes inline
The side-by-side description/meta split becomes chips row → inline key-value pairs → description. Same fields, read top to bottom.

      
Filter stays visible
bugs still lives in the crumb and the footer still offers f bugs ✕ — visible state survives narrowing before convenience keys do.

    


    

      
4 · Detail — the rail folds into "facts"

      
Facts panel up top
The right rail's people/meta becomes a two-pair-per-row grid at the top of the scroll, with the workflow strip inside it. One x keybinding folds it to a single line once you've read it.

      
Links and children merge
At this width, "linked" is one panel: blocks/relates rows and child rows share the relation-coloured prefix column. Same colours as the landscape set.

      
Reading column stays honest
Description and activity keep the full width — the one thing a narrow terminal is good at is a comfortable single reading column.

    


    

      
5 · Board — a column pager, not a squeeze

      
This is the DS-137 fix
"Board lanes collapse on narrow terminals" — because four columns can't share 84 cells. Below the breakpoint the board shows one status column at a time: the tab strip is the map, ←/→ pages columns, and cards get full width.

      
Neighbour peek
The selected card shows ◂ To Do 2 · In Review 0 ▸ — where this epic's other issues sit in the adjacent columns — so paging never feels blind.

      
Lanes still group by epic
Swimlane headers survive with "1 here · 3 total" counts, and lanes empty in the current column collapse into one ghost line.

    


    

      
6 · Command palette — width-proof by design

      
The modal just narrows
The palette is a list; it degrades by truncating labels, never by dropping actions. At any width it remains the complete map of the keyspace.

      
More valuable when narrow
Narrow terminals show fewer footer hints, so the palette carries more of the discovery load — the two features are designed as a pair.

    

  

  


  
Companion to ui-refresh.html (the landscape set) — same tokens, same widgets, recomposed for < 90 columns. The adaptive rules are all cell-count driven and map to plain ratatui Layout switches: strips instead of rails, a paged board instead of a grid, and a fixed column-drop order for lists. Jax scales down gracefully and remains extremely happy. 🦦🍁
