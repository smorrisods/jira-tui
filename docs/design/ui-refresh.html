Design proposal · docs/design

jira-tui · UI refresh — six screens, one calmer terminal

The app has grown a lot of capability since the first ratatui pass — tree view, boards, comments, links, history — and the chrome hasn't kept up. This proposal keeps the cyan / magenta identity and the three-region layout, and adds three things: structured rows instead of packed strings, grouped key hints instead of one long footer, and a third accent — maple, borrowed from Jax's leaf — used only for "you are here": selection, focus, and Jax moments.

Jax is not only retained, Jax is promoted. All six scenes stay; the companion gains a mood line and reacts to what you do. Click him below.

description

  • Repro: open DS-2 in demo mode, press e, save unchanged

  • Suspect: indent tracking in src/adf/markdown.rs (tab-selected link — ⏎ opens)

Acceptance criteria

  • Three-level list round-trips byte-identical ADF
  • Fixture added to tests/render.rs
  • Preview diff shows no change for a no-op edit

1 · Welcome — Jax makes the pitch

      

        
Three doors, one recommended
The current welcome lists three flat rows. Making them cards with equal weight — and one recommended maple border — answers "which one do I press?" without reading three descriptions. Demo mode is explicitly framed as never going away.

        
Security note where the decision happens
The 0600 token promise moves into the "Connect" card itself, at the moment the person is deciding whether to paste a token.

        
Jax unchanged
Same box body, same blink, same swappable 🍁/🍂. He earned his spot; the refresh just gives him room to breathe and a real headline instead of a caption.

      

    


    

      
2 · Home — a dashboard, not just a split

      

        
Breadcrumb replaces packed title strings
Today the list title concatenates view, sort, filter, and count into one string. The header gets a proper crumb (My Work › Home) and the panel title keeps only sort/filter/count as muted metadata.

        
Glance stats get bars
Counts alone don't scan; a 4px bar under each stat gives proportion at a glance. Blocked stays ember and only draws attention when non-zero.

        
Grouped footer hints
The current footer is one long dot-separated string. Grouping hints under tiny NAV / VIEW / GO labels lets the eye jump to the right cluster, and ? all keys takes overflow pressure off.

        
Recent issues
A small "recent" panel (backed by the existing history stack) makes the home screen a place you return to, not just pass through.

      

    


    

      
3 · List — columns you can actually scan

      

        
Real columns with headers
Key, type, status, summary, assignee, updated become aligned columns with a one-line header. Status moves from coloured text to a tinted chip — same semantic colours, far easier to scan down.

        
Maple selection bar
Selection changes from a magenta ▌ glyph to a maple left bar + tint across the whole row — the row reads as selected even in peripheral vision, and magenta stays free for panel identity.

        
Quick view grows a meta column
Instead of re-rendering the full detail into a half-height pane, quick view splits: description excerpt left, compact field grid right. Full detail remains one ⏎ away.

        
Active filter is visible and dismissable
filter: bugs appears in the crumb and the footer shows f filter: bugs ✕ — state you can see is state you can trust.

      

    


    

      
4 · Detail — content left, facts right

      

        
Two columns end the long scroll
Today everything is one scrolling paragraph stack. Description and activity keep the wide column; people, workflow, labels, links, and children move to a fixed right rail so they're visible without scrolling past the description.

        
Workflow strip becomes a mini-map
The transition strip keeps its → shape but each status gets a dashed outline, with the current one solid orchid — it reads as "you are here on a track", and t still opens the picker.

        
Comments as anchored cards
Each comment gets a left rule (maple for your own), author and time on one line, and n/p jumping preserved. ADF rendering is untouched — this is layout only.

        
Acceptance criteria as checkboxes
When the mapped AC field parses as a list, render ☐/☑ — the field-mapping feature already knows which field this is.

      

    


    

      
5 · Board — cards instead of a text grid

      

        
Bordered cards
Cells become small bordered cards: priority + key + type chip on top, summary, assignee + age on the foot. The selected card swaps the Rgb(40,40,80) background for the maple bar used everywhere else.

        
Columns own their colour
Column headers keep status colours and gain counts and an optional amber WIP marker — the only place amber appears on this screen, so it means something.

        
Lanes stay, empties collapse
Epic swimlanes remain the organizing idea; empty cells render a dashed ghost and fully-done lanes can collapse behind pgdn, which fixes most of the narrow-terminal pain.

      

    


    

      
6 · Command palette — proposed, not yet in the app

      

        
One key to rule the chrome
The footer can't keep absorbing new features. A ctrl-k palette lists every action with its key beside it — it's discoverable help that also executes, and it lets footer hints stay short.

        
Context sections
Actions group by scope: on DS-129 (transitions, assign, copy), view (sort, filter, tree), app (refresh, Jax, mouse). Transitions listed inline means "move to Done" is two keystrokes from anywhere.

        
Teaches the fast path
Every row shows its direct keybinding, so the palette trains you out of needing it — the same philosophy as the current footer, with room to grow.

      

    

  


  
Built as static HTML mockups for docs/design/ — the terminal frame is one deliberate dark world (it's a terminal), while this page adapts to your theme. Everything shown maps to existing ratatui widgets: panels are Block + Rounded, chips are the existing chip() helper with tinted backgrounds, bars and cards are line-level styling. The only new capability proposed is the command palette. Jax approves this message. 🦦🍁
